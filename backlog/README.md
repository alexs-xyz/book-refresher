# Backlog Structure

This backlog turns the product and architecture docs into Codex-sized execution units.

## How to use
- Work one slice at a time.
- For anything non-trivial, create or update a `plans/*.md` file first.
- Keep slices small enough that Codex can implement, verify, and hand back a reviewable diff.
- Do not merge multiple epics into one prompt unless the tasks are tightly coupled.
